# Windows RTX 4080 Training Runbook

This runbook trains and evaluates the DistilBERT reranker for the Ask James chatbot on a Windows computer with an NVIDIA GPU.

Run commands in **PowerShell**, from the repository root. This guide deliberately uses the project virtual environment's Python executable after setup so a later automatic dependency sync does not accidentally replace the CUDA-enabled PyTorch build.

## 0. Before leaving the Mac

Commit and push the project source, curated input data, tests, and lockfile. Do **not** commit the trained model, `.env`, or generated model artifacts.

The Windows checkout must contain at least:

```text
backend/
data/chunks.json
data/qa_pairs.jsonl
data/unanswerable_questions.jsonl
kb_extra/
tests/
pyproject.toml
uv.lock
```

The following files are generated by the commands below and are intentionally machine-local:

```text
data/bm25_index.json
data/train.jsonl
data/val.jsonl
data/test.jsonl
data/eval_results.json
data/training_history.json
models/reranker/
```

## 1. Windows prerequisites

### 1.1 Open PowerShell

Use normal PowerShell; administrator access is not normally needed.

### 1.2 Verify the NVIDIA driver and GPU

```powershell
nvidia-smi
```

Expected result: a table containing the RTX 4080 and an NVIDIA driver version. If PowerShell says `nvidia-smi` is not recognized, install or update the NVIDIA driver before continuing.

### 1.3 Install Git, Python 3.12, and uv if needed

Check whether they already exist:

```powershell
git --version
py -3.12 --version
uv --version
```

If a command is missing, install it with Windows Package Manager:

```powershell
winget install --id Git.Git -e
winget install --id Python.Python.3.12 -e
winget install --id astral-sh.uv -e
```

Close and reopen PowerShell after installation so the updated `PATH` is available.

### 1.4 Check available disk space

Leave at least 10 GB free. Python packages, CUDA-enabled PyTorch, Hugging Face model downloads, cache files, and the trained model all use disk space.

```powershell
Get-PSDrive C
```

## 2. Get the project on Windows

### Option A: First checkout on this computer

```powershell
cd $HOME\Documents
git clone https://github.com/Cheezecats/about-me-bot-website.git
cd about-me-bot-website
```

### Option B: The repository already exists

```powershell
cd $HOME\Documents\about-me-bot-website
git status
git pull --ff-only origin main
```

`git status` should be clean before pulling. If it is not clean, do not discard local work; commit, stash, or resolve it first.

### Confirm the training inputs arrived

```powershell
Test-Path data\chunks.json
Test-Path data\qa_pairs.jsonl
Test-Path backend\training\train_reranker.py
Get-Content data\qa_pairs.jsonl | Measure-Object -Line
```

Expected result: each `Test-Path` command returns `True`, and the QA dataset has approximately 203 lines.

## 3. Create the Python environment

From the repository root:

```powershell
uv python install 3.12
uv sync --frozen --extra ml --extra dev
```

This installs project dependencies from `pyproject.toml` and `uv.lock`, including PyTorch, Transformers, pytest, and the training dependencies.

### Important: replace the default Windows CPU PyTorch build with CUDA PyTorch

The current project lockfile resolves ordinary PyPI PyTorch for Windows, which is CPU-only. For RTX 4080 training, install a CUDA build after the standard sync:

```powershell
uv pip install --reinstall torch --torch-backend=auto
```

`--torch-backend=auto` asks uv to detect the installed NVIDIA driver and select a compatible PyTorch CUDA build.

If automatic detection fails, use a PyTorch CUDA backend that matches the official PyTorch selector and your driver, for example:

```powershell
uv pip install --reinstall torch --torch-backend=cu126
```

Do not guess a CUDA version if this fails. Visit the official PyTorch selector, choose **Windows / Pip / Python / CUDA**, and use the offered command or ask for help with the `nvidia-smi` output.

## 4. Verify the environment before training

Use the project environment's Python directly:

```powershell
.venv\Scripts\python.exe -c "import sys, torch, transformers; print(sys.version); print('torch:', torch.__version__); print('transformers:', transformers.__version__); print('cuda available:', torch.cuda.is_available()); print('gpu:', torch.cuda.get_device_name(0) if torch.cuda.is_available() else 'CPU ONLY')"
```

Required result:

```text
cuda available: True
gpu: NVIDIA GeForce RTX 4080
```

If it says `False` or `CPU ONLY`, **stop here**. Training would work but be much slower and would not use the 4080.

### Run the test suite

```powershell
.venv\Scripts\python.exe -m pytest -v
```

Expected result: all tests pass. The suite currently includes BM25, privacy, QA integrity, and reranker-scoring tests.

### Optional: capture the environment for your IA log

```powershell
nvidia-smi
.venv\Scripts\python.exe -c "import torch, transformers; print('torch=', torch.__version__); print('transformers=', transformers.__version__); print('cuda=', torch.version.cuda)"
```

## 5. Build the train, validation, and test datasets

```powershell
.venv\Scripts\python.exe -m backend.training.build_dataset
```

Expected output is similar to:

```text
[build_dataset] questions: train=140 val=30 test=33
[build_dataset] rows:      train=700 val=150 test=33
```

The exact split counts may change if the committed QA data changes. Confirm that:

- the three question counts are non-zero;
- training has substantially more questions than validation and test;
- pair rows contain one positive and four negatives per training question;
- no error reports a missing positive chunk ID.

The command creates:

```text
data/train.jsonl
data/val.jsonl
data/test.jsonl
```

## 6. Optional baseline: BM25 parameter grid search

Run this before training if you want to record a BM25 baseline:

```powershell
.venv\Scripts\python.exe -m backend.training.bm25_grid_search
```

It writes:

```text
data/bm25_grid_search.json
```

Important: do not change `BM25_K1` or `BM25_B` based on a grid search that uses the full QA dataset, because that would use held-out test questions during parameter selection. For a final IA experiment, select parameters on training/validation data only, then evaluate once on the test data.

## 7. Train the DistilBERT reranker

Ensure the computer is plugged into power and prevent sleep during the run.

```powershell
.venv\Scripts\python.exe -m backend.training.train_reranker
```

Expected early output:

```text
[train] device=cuda train_questions=140 val_questions=30 candidates_per_question=5
```

If this says `device=cpu`, stop and return to the CUDA verification step.

The first run downloads `distilbert-base-uncased` from Hugging Face. Internet access is required for this download unless the model is already cached.

Expected epoch output resembles:

```text
[train] epoch 1/8 train_loss=... train_p@1=... train_mrr=... val_loss=... val_p@1=... val_mrr=...
[train] epoch 2/8 ...
```

Training uses listwise cross-entropy: each question is trained against its one positive and four negative chunks as a ranking group. This avoids treating the 80% negative-class proportion as a meaningful accuracy result. The best model is selected by validation MRR, and training may stop after two non-improving validation epochs.

On success, verify these outputs exist:

```powershell
Test-Path models\reranker\config.json
Test-Path models\reranker\model.safetensors
Test-Path data\training_history.json
Get-ChildItem models\reranker
Get-Content data\training_history.json
```

Some Transformers versions write `pytorch_model.bin` instead of `model.safetensors`; either is valid.

## 8. Evaluate the configured reranker backend

```powershell
.venv\Scripts\python.exe -m backend.training.evaluate
```

This evaluates the production retrieval route:

```text
question -> BM25 top 10 -> configured reranker -> ranking metrics
```

It writes:

```text
data/eval_results.json
```

Display the saved result:

```powershell
Get-Content data\eval_results.json
```

Record at least:

- `bm25_candidate_recall` — ceiling of what the reranker can recover;
- `precision_at_1` — correct labeled chunk ranked first;
- `hit_at_3` — correct labeled chunk appears in top three;
- `mrr` — overall rank quality;
- `n_questions` — test-set size.

Do not describe these metrics as final product accuracy. They measure chunk ranking, not whether Qwen produced a correct grounded answer.

Compare the reranker with the `bm25_precision_at_1`, `bm25_hit_at_3`, and `bm25_mrr` fields in the same output. Do not deploy it merely because training completed: it must improve on the BM25 baseline on the held-out test set.

## 9. Calibrate the refusal threshold

This step needs the curated unanswerable-question file:

```powershell
Test-Path data\unanswerable_questions.jsonl
.venv\Scripts\python.exe -m backend.training.calibrate_threshold
Get-Content data\threshold_calibration.json
```

The calibration script selects a threshold using answerable validation questions and 70% of the unanswerable questions. It then reports the selected threshold on the untouched answerable test questions and the remaining unanswerable holdout questions. This prevents threshold selection from leaking test-set information.

```text
data/threshold_calibration.json
```

The output includes `recommended_for_deployment`. Do not change `CONFIDENCE_THRESHOLD` unless this is `true`, the held-out rates are acceptable, and the reranker improves on BM25. A result of `false` means there is no threshold that is both sufficiently safe and sufficiently useful for this model.

## 10. Optional ablation study

The ablation script compares:

```text
BM25 only
BM25 + zero-shot MiniLM cross-encoder
BM25 + fine-tuned DistilBERT
```

The zero-shot cross-encoder is included in the project's `ml` extra. Before running the ablation, install the reproducible extras:

```powershell
uv sync --extra ml --extra dev
```

Then run:

```powershell
.venv\Scripts\python.exe -m backend.training.ablation_study
Get-Content data\ablation_results.json
```

This is valuable IA evidence because it tests whether fine-tuning actually improves the result over simpler alternatives.

## 13. Common failures

### `ModuleNotFoundError: No module named 'transformers'`

The wrong environment is active or dependencies were installed using the incomplete `requirements.lock`.

```powershell
uv sync --frozen --extra ml --extra dev
.venv\Scripts\python.exe -c "import transformers; print(transformers.__version__)"
```

### Training prints `device=cpu`

CUDA PyTorch is not installed or the NVIDIA driver is unavailable.

```powershell
nvidia-smi
.venv\Scripts\python.exe -c "import torch; print(torch.__version__); print(torch.cuda.is_available()); print(torch.version.cuda)"
uv pip install --reinstall torch --torch-backend=auto
```

### `OSError` or network failure while loading `distilbert-base-uncased`

The first training run needs internet access to download the base model. Restore network access and rerun. Once downloaded, Hugging Face normally caches it locally.

### `positive_chunk_id not found in chunks`

The QA file and chunks file are from different commits.

```powershell
git status
git log -1 --oneline
git pull --ff-only origin main
```

Then rebuild the generated datasets.

### GPU out-of-memory error

The current batch size is 16 question groups (80 question/chunk pairs) and should be comfortable on an RTX 4080. If it fails, reduce `TRAIN_BATCH_SIZE` in `backend/config.py` to 8, rebuild nothing, and rerun training. Record the changed setting in the experiment log.

### `sentence_transformers` error during ablation

Install the optional dependency:

```powershell
uv pip install sentence-transformers
```

### Tests fail because QA evidence changed

Inspect the reported question and chunk. Correct the QA mapping or, if the KB intentionally changed, update the evidence assertion in `tests/test_qa_pairs.py` only after verifying the new source text manually.

